# Text-extractor-S3-fetching-

This project is a PDF-to-database pipeline for lab reports: `main.py` is the entrypoint that triggers `s3_batch.py`, which pulls new PDFs from S3, retries failed ones, parses patient names, calls the extractor, inserts rows into PostgreSQL, and tracks processed S3 keys in `processed_files.json`; `scheduler.py` runs that batch continuously on a time interval; the `extractor/` package contains the modular parsing logic (`core.py`, `column.py`, `row_processor.py`, `utils.py`, `__init__.py`) that converts report tables into structured lab-result DataFrames with cleaned units/ranges and min/max bounds; `parser/pdf_extractor.py` is an older monolithic extractor version; `db/db_config.py`, `db/insert_data.py`, and `db/report_audit.py` handle DB connection, lab result inserts, and success/failure audit/retry state; `requirements.txt` lists dependencies; `.env` stores AWS/S3/local-path config; `csv/` and `output.csv` are sample/output datasets; `reports/` stores downloaded PDF files; `old/` contains legacy batch/scheduler scripts; and `.gitignore` excludes virtual envs, caches, env files, PDFs, and generated data.
